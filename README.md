# tanabata

現場監督向けの小さなDXツール集です。社内Box上のHTMLを開いて使う前提で、サーバーやインストールなしで動く静的Webアプリとして育てます。

## 目的

- 現場で毎日発生する定型作業を、短時間で確実に処理できるようにする
- 会社PCの制約が強くても使えるように、HTML/CSS/JavaScriptだけで構成する
- Copilot 365には直接組み込まず、必要な文章やプロンプトをコピーして渡せる形にする
- 最初は小さく作り、現場で使いながらツールを追加していく

## 配布・実行方針

社内Boxで配る場合は、Box DriveでPCのファイルエクスプローラー上に表示されているフォルダから開く運用を基本にします。

- 推奨: エクスプローラー上のBox Driveフォルダから`index.htm`を直接クリックして開く
- 代替: Boxからフォルダごとダウンロードして`index.htm`を開く
- 非推奨: BoxのWebプレビュー上でHTMLを直接動かす

Box Driveで同期されたファイルをエクスプローラーから直接クリックする方式なら、通常のローカルHTMLとしてブラウザで開けます。相対パスのCSS/JS/画像も読み込みやすく、複数ファイル構成にもできます。BoxのWebプレビュー上で開く場合は、HTML/JavaScriptの実行や相対ファイル参照が制限される可能性があります。

## CSS・JavaScriptの扱い

外部CSSや外部CDNは、技術的には使える場合がありますが、このプロジェクトでは原則使いません。

理由:

- 会社PCや社内ネットワークでCDNがブロックされる可能性がある
- オフライン、現場回線不安定、Box同期直後でも動く方が強い
- 外部通信をしない方が社内展開時に説明しやすい
- バージョン差分やライセンス管理が複雑になる

基本方針:

- CSSは`assets/css/`に置く
- JavaScriptは`assets/js/`に置く
- 画像やアイコンは`assets/img/`に置く
- ライブラリが必要な場合は、CDNではなく`vendor/`に同梱する
- データはJSONを`fetch()`で読むより、JSファイルとして読み込む方が安全

`file://`でHTMLを開く場合、ブラウザによっては`fetch("./data.json")`やES Modulesが制限されます。そのため、最初はクラシックな`<script src="...">`構成で作ります。

## 想定フォルダ構成

```text
tanabata/
  README.md
  index.htm
  app/
    index.html
    assets/
      css/
        styles.css
      js/
        app.js
```

利用者はルートの`index.htm`を開きます。実体は`app/index.html`に置き、画面内で各ツールを切り替える構成にします。ページ遷移やビルドツールは使わず、コピーして配れば動く形を優先します。

## 技術方針

- フレームワークなしのHTML/CSS/JavaScriptで作る
- すべてローカル完結にして、外部APIへ送信しない
- Edge/Chromeの通常ブラウザで動くことを優先する
- 入力値、計算式、結果を画面上で確認しやすくする
- 計算ツールには「前提」「式」「注意」を表示する
- 個人設定や履歴は`localStorage`に保存する
- 共有したいテンプレートは`templates.js`に固定データとして持たせる
- 会社標準や現場ルールに関わる計算は、必ず人が確認する

## Copilot 365との付き合い方

ローカルHTMLからCopilot 365を直接呼び出す設計は、認証、管理者設定、Microsoft 365側の制限が絡むため初期方針には入れません。

代わりに、次の使い方を想定します。

- ツールで文章や表を整形する
- Copilotに渡すプロンプトを自動生成する
- ボタンでクリップボードへコピーする
- Copilot 365のチャットへ貼り付ける

これなら社内制約に引っかかりにくく、現場でも運用しやすいです。

## 初期ツール2: スリーブ梁底計算ツール

### 目的

梁底を基準に、スリーブの芯高さや下端余裕を素早く確認するための補助ツールです。

### 注意

このツールは現場確認用の計算補助です。設計図、構造図、社内基準、監理者・設計者確認を置き換えるものではありません。梁貫通、かぶり、補強筋、梁成、構造上の制限が絡むため、計算式は現場の標準に合わせて確定します。

### 初期入力案

- 階高（FL）
- 梁スリーブ芯FL
- 梁成
- 下端フカシ
- スラブ天端レベル

### 初期出力案

- 梁底からスリーブ芯までの距離
- `梁底+480mm`形式のコピー用テキスト
- 梁底FL
- スリーブ芯FL
- 保存日時付きの保存履歴

### 初期式案

```text
梁底FL = 階高（FL） - 梁成 - 下端フカシ + スラブ天端レベル
梁底からスリーブ芯までの距離 = 梁スリーブ芯FL - 梁底FL
```

現場運用で確認したいこと:

- 下端フカシを梁底FLから引く扱いで正しいか
- スラブ天端レベルの正負方向が現場表記と合っているか
- 端数処理を1mm、5mm、10mmのどれにするか
- NG判定条件を入れるか

## 追加候補

簡単にツール化しやすく、現場への効果が出やすい候補です。

1. 写真コメント整形ツール
   - 撮影対象、場所、作業内容、是正前後を入力して写真台帳コメントを生成する。

2. 指摘事項・是正依頼文ツール
   - 指摘内容、場所、期限、依頼先を入れて、角が立ちにくい依頼文に整える。

3. 日報メモ整形ツール
   - 箇条書きメモから、日報に貼れる文章へ整形する。

4. 風量・風速・ダクトサイズ計算
   - 風量とダクト寸法から風速を出す。丸ダクト、角ダクトの簡易確認に使う。

5. ドレン勾配計算
   - 距離と勾配から必要落差を出す。逆に落差から確保勾配も確認する。

6. 配管・ダクト支持ピッチ確認メモ
   - 社内標準や現場ルールを登録し、径や材質から確認メモを出す。

7. 搬入・揚重チェックリスト
   - 機器名、寸法、重量、搬入経路、立会者、必要工具をチェックする。

8. 工程連絡テンプレート
   - 明日作業、立会依頼、入場調整、停電・断水・火気使用などの連絡文を生成する。

9. 単位換算ツール
   - mm/m、Pa/mmAq、m3/h/L/s、kW/kcal/hなど、設備で使う単位を変換する。

10. 現場用チェックリスト
    - スリーブ、インサート、墨出し、区画貫通、保温、試運転などの確認項目をテンプレ化する。

## 現在の実装

- ルートの`index.htm`から起動
- 実体は`app/index.html`
- シンプルでコンパクトなレスポンシブUI
- コピペツール
  - 文字の追加
  - カードクリックでコピー
  - 編集
  - `×`ボタンで削除
  - ドラッグで並び替え
  - `localStorage`保存
- スリーブ梁底計算ツール
  - 階高（FL）、梁スリーブ芯FL、梁成、下端フカシ、スラブ天端レベルを入力
  - 数値欄は半角数字へ自動変換
- 空欄は0として計算
- 各入力欄の`×`ボタンで0に戻す
- リセットボタンで全入力を0に戻す
  - 入力と同時に結果を更新
  - `梁底+480mm`形式でコピー
  - 入力値を`localStorage`保存
  - 保存日時付きの履歴表示
  - 履歴から復元・削除
